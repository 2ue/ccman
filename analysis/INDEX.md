# ccman 源码驱动项目分析索引

> 本索引基于 **源码事实**(packages/core, packages/cli, packages/desktop 等)整理,所有结论均以代码为主,文档(`./docs`)仅作为对照与背景参考。

## 文档一览

- `analysis/A-项目概述与定位.md`  
  从源码实际行为出发,给出 ccman 的项目目标、边界与真实特性,并对 README 中的描述进行“以代码为准”的校准。

- `analysis/B-技术架构与代码结构.md`  
  以代码结构为唯一依据,梳理 Core/CLI/Desktop 三层架构,说明调用关系、依赖方向,以及关键设计模式(例如 ToolManager + Writers + Sync)。

- `analysis/C-功能模块与数据流.md`  
  针对服务商管理、WebDAV 同步、MCP 管理、导入导出、Claude 清理等功能,用“入口 → Core API → 外部文件/网络”的方式画出真实数据流。

- `analysis/D-工程质量与缺陷分析.md`  
  聚焦工程实践:类型设计、错误处理、环境隔离、文件 IO、安全性等;所有缺陷与风险点都对应具体源码位置,而不是停留在文档层面的设想。

- `analysis/E-优化与演进建议.md`  
  在前面分析基础上,提出可执行的改进建议和演进路线,每一条建议都标明关联的代码文件/模块,方便后续迭代时直接定位实现处。

> 建议阅读顺序: A → B → C → D → E  
> 如果你已经非常熟悉项目背景,可以直接从 `D-工程质量与缺陷分析.md` 和 `E-优化与演进建议.md` 开始,再按需回看其它章节。
